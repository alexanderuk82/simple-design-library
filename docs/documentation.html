Getting Started

Installation

Get Simple Design System up and running in your project in minutes. Choose your preferred package manager or use our CDN for quick prototyping.

Requirements

Simple DS works with any modern framework or vanilla HTML/CSS. No dependencies required.

Package Manager

bash 
npm install simple-design-system
bash 
yarn add simple-design-system
bash 
pnpm add simple-design-system
html 
<!-- CSS only -->
<link rel="stylesheet" href="https://unpkg.com/simple-design-system/dist/simple-ds.css">

<!-- With JavaScript components -->
<script src="https://unpkg.com/simple-design-system/dist/simple-ds.js"></script>
Getting Started

Quick Start

Start using Simple DS in your project with just a few lines of code.

Import the CSS

javascript 
// In your main JS/TS file
import 'simple-design-system/css'

// Or import specific modules
import 'simple-design-system/css/tokens.css'
import 'simple-design-system/css/components.css'

Use the Components

html 
<!-- Button Component -->
<button class="sds-btn sds-btn--primary">
  Get Started
</button>

<!-- Card Component -->
<div class="sds-card">
  <div class="sds-card__header">Card Title</div>
  <div class="sds-card__body">Content here</div>
</div>

Customization

Theming

Customize Simple DS to match your brand. Override CSS variables to change colors, fonts, spacing, and more throughout the entire system.

Override Colors

css 
/* Your custom theme */
:root {
  /* Primary accent color */
  --accent: oklch(0.65 0.25 150);
  --accent-light: oklch(0.75 0.20 150);
  --accent-dark: oklch(0.55 0.25 150);

  /* Or use hex/rgb/hsl */
  --accent: #10b981;
}

Override Fonts

css 
/* Import your fonts first */
@import url('https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;500;700&display=swap');

:root {
  --font-display: 'Space Grotesk', sans-serif;
  --font-body: 'Space Grotesk', sans-serif;
}

Guidelines

Accessibility

Simple DS is built with accessibility in mind. All components meet WCAG 2.1 AA standards.

Key Features

Color Contrast

All color combinations meet WCAG 2.1 AA contrast requirements (4.5:1 for normal text, 3:1 for large text).

Keyboard Navigation

All interactive elements are fully keyboard accessible with visible focus states.

Screen Reader Support

Proper ARIA labels and semantic HTML ensure compatibility with assistive technologies.

Reduced Motion

Respects prefers-reduced-motion for users who are sensitive to animations.

Guidelines

Best Practices

Tips for getting the most out of Simple Design System.

Use Design Tokens

Always use CSS variables instead of hard-coded values. This ensures consistency and makes theming easy.

css 
/* Good */
.my-component {
  padding: var(--space-md);
  color: var(--fg);
  background: var(--bg-subtle);
}

/* Avoid */
.my-component {
  padding: 16px;
  color: #1f1f1f;
  background: #f5f5f5;
}
